Agent System

An async generator-based runtime that orchestrates LLM conversations with tool calling, streaming, and stuck detection.

Built-in Agents

Three agents are registered at startup from src/core/agents/builtin/:

AgentDescription
researchDeep research with web search and content extraction
browserBrowser automation via browser tools
chatGeneral conversation with optional tool access

Runtime Loop

The runtime (src/core/agents/runtime.ts) is an async generator yielding events as the agent progresses:

runAgent(config) → AsyncGenerator<AgentEvent>

  1. Get default provider from providerRegistry
  2. Call agent.preloadContext(context) if defined
  3. Build system prompt via agent.buildSystemPrompt(context)
  4. Yield { type: 'system_prompt', content }

  ┌─ Tool Call Loop (max 25 iterations) ──────────────┐
  │  5. provider.chat(messages, tools, 'auto')         │
  │  6. Fallback: flatten tool messages on error        │
  │  7. Fallback: extract tool calls from text (JSON)   │
  │  8. Stuck detection: same call 3× → stop            │
  │  9. Split into parallel vs sequential batches        │
  │  10. Execute parallel batch (Promise.all)            │
  │  11. Execute sequential batch (one at a time)        │
  │  12. Yield tool_call, tool_result, action events     │
  └─────────────────────────────────────────────────────┘

  13. Final streaming call (chatStream, no tools)
  14. Yield token, usage, done events

Agent Events

Events consumed by the frontend via electronAPI.onAgentEvent(cb):

TypeDescription
system_promptSystem prompt built for this session
tool_callAgent is calling a tool (name, input)
tool_resultTool execution result
actionUI side effect (navigate, open tab, etc.)
web_sourcesURLs referenced during research
tokenStreaming text token from final response
usageToken usage statistics
doneAgent session complete
_stoppedAgent stuck (same tool call repeated 3 times)

Tool Execution

When the LLM returns tool calls, the runtime splits them based on the tool's sequential flag:

  • Parallel batch — tools without sequential: true run concurrently via Promise.all
  • Sequential batch — tools with sequential: true run one at a time
// ToolContext passed to each tool during agent execution
const context: ToolContext = {
  googleId,
  signal: abortController.signal,
  onProgress: (data) => emit({ type: 'progress', data }),
  emitAction: (action) => emit({ type: 'action', ...action }),
  waitForResult: (requestId, timeoutMs) =>
    waitForFrontendResult(requestId, timeoutMs),
  browserDom,
  documentText,
};

Action Result Protocol

Some tools need input from the frontend. The action result protocol enables async round-trips:

  1. Tool calls context.waitForResult(requestId) — returns a Promise
  2. Runtime emits an action event to the frontend
  3. Frontend calls electronAPI.agentActionResult(requestId, result)
  4. Main process resolves the Promise via resolveActionResult()

Default timeout: 15 seconds.

Frontend Integration

// Start an agent session
const sessionId = await electronAPI.agentStart({
  agent: 'research',
  messages: [{ role: 'user', content: 'Find papers on attention' }],
  toolsEnabled: true,
});

// Listen for streaming events
electronAPI.onAgentEvent((event) => {
  switch (event.type) {
    case 'token':
      appendToChat(event.content);
      break;
    case 'tool_call':
      showToolIndicator(event.name);
      break;
    case 'done':
      finishChat();
      break;
  }
});

// Cancel if needed
electronAPI.agentCancel(sessionId);
