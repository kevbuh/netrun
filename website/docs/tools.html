Tool System

33 tools across 8 categories, registered in a central registry with Zod validation and a middleware chain.

Tool Registry

The ToolRegistry singleton manages all tools. Tools are registered at startup via registerAllTools() in src/core/tools/index.ts.

MethodDescription
register(tool)Register a tool (throws on duplicate name)
use(middleware)Add middleware (first = outermost)
get(name)Look up a tool by name
getByCategory(category)Get all tools in a category
getByAccess(access)Get tools by access level
execute(name, input, context)Execute through the middleware chain
toToolDefinitions(access?)Export as OpenAI-compatible JSON Schema
names()List all tool names
all()Get all tools
categories()List all categories

Tool Interface

interface Tool<TInput, TOutput> {
  name: string;
  description: string;
  category: string;
  access: ToolAccess[];          // 'agent' | 'mcp' | 'ui'
  parameters: z.ZodType<TInput>; // Zod schema for validation
  sequential?: boolean;          // true = cannot run in parallel
  execute(input: TInput, context: ToolContext): Promise<ToolResult<TOutput>>;
}

ToolContext

interface ToolContext {
  googleId?: string;
  signal?: AbortSignal;
  onProgress?: (data: unknown) => void;
  emitAction?: (action: { type: string }) => void;
  waitForResult?: (requestId: string, timeoutMs?: number) => Promise<unknown>;
  browserDom?: string;
  documentText?: string;
}

ToolResult

interface ToolResult<T> {
  success: boolean;
  data?: T;
  error?: string;
}

ToolAccess

  • 'agent' — available to AI agents via tool calling
  • 'mcp' — available via MCP protocol
  • 'ui' — available to frontend code

Middleware Chain

Two middleware are registered at startup:

  1. Logging (outermost) — logs tool name, duration, and result
  2. Timeout (inner) — enforces execution time limits
type ToolMiddleware = (
  tool: Tool,
  input: unknown,
  context: ToolContext,
  next: () => Promise<ToolResult>
) => Promise<ToolResult>;

// Chain built with reduceRight — first registered = outermost
const chain = middlewares.reduceRight(
  (next, mw) => () => mw(tool, input, context, next),
  coreExecute
);

Execution Flow

toolRegistry.execute('webSearch', { query: 'test' }, context)
  1. Look up tool by name       → found or error
  2. Validate input via Zod     → parsed or error
  3. Build middleware chain      → logging → timeout → execute
  4. Run chain                   → ToolResult
  5. Return result               → { success, data } or { success: false, error }

Adding a Tool

Create a file in the appropriate category folder under src/core/tools/:

import { z } from 'zod';
import { Tool } from '../types';

export const myTool: Tool<{ query: string }, string> = {
  name: 'myTool',
  description: 'Does something useful',
  category: 'search',
  access: ['agent', 'ui'],
  parameters: z.object({
    query: z.string().describe('The search query'),
  }),
  async execute(input, context) {
    const result = await doSomething(input.query);
    return { success: true, data: result };
  },
};

Then register it in src/core/tools/index.ts:

import { myTool } from './search/my-tool';
toolRegistry.register(myTool);
