IPC & Preload

The context bridge exposes 100+ methods on window.electronAPI, connecting the renderer to the main process and core backend.

How It Works

Electron's context isolation means the renderer has no direct access to Node.js APIs. The preload.js script bridges this gap using contextBridge.exposeInMainWorld:

// electron/preload.js
contextBridge.exposeInMainWorld('electronAPI', {
  // Request/response (async)
  toolExecute: (name, input, ctx) =>
    ipcRenderer.invoke('tool:execute', name, input, ctx),

  // Push events (subscribe)
  onAgentEvent: (cb) =>
    ipcRenderer.on('agent:event', (_e, data) => cb(data)),

  // DB shortcut — prefixes channel with 'db:'
  dbQuery: (channel, ...args) =>
    ipcRenderer.invoke('db:' + channel, ...args),
});

Two communication patterns:

  • ipcRenderer.invoke(channel, ...args) — async request/response. Returns a Promise.
  • ipcRenderer.on(channel, callback) — subscribe to push events from the main process.

Routing Flow

Frontend                    Preload                     Main / Core
────────                    ───────                     ───────────
electronAPI.toolExecute()
  → ipcRenderer.invoke('tool:execute', ...)
                              → ipcMain.handle('tool:execute')
                                                          → toolRegistry.execute()
                                                          ← { success, data }
                              ← result
  ← await resolves

IPC Handler Modules

Backend-side handlers live in src/core/ipc/, one file per subsystem. Each module registers ipcMain.handle calls:

// Typical pattern (src/core/ipc/tools-providers.ts)
import { ipcMain } from 'electron';
import { toolRegistry } from '../tools/registry';

export function registerToolsProvidersIPC() {
  ipcMain.handle('tool:execute', (_event, name, input, context) => {
    return toolRegistry.execute(name, input, context);
  });

  ipcMain.handle('tool:list', () => {
    return toolRegistry.names();
  });
}

All 15 IPC modules are registered during initCore(). See Architecture Overview for the full list.

Main Process IPC

Some handlers live directly in electron/main.js rather than the core backend, handling Electron-specific features:

  • Adblock — 6 channels for engine control
  • Privacy — 12 channels for toggle get/reset/set
  • Window — position, cursor nudge, print
  • Screen capture — screen regions and webview contents
  • Auth — Keychain via safeStorage
  • Password manager — encrypted credential storage
  • File dialogs — save, open, temp file handling
