Reactivity

Fine-grained signal-based reactivity inspired by SolidJS. No virtual DOM — dependencies are tracked automatically and updates are surgical.

State

A writable signal that holds a single value. Reading .value inside an Effect or Computed automatically subscribes to changes.

const count = State(0);
const name = State('Alice', { name: 'userName' });

// Read
console.log(count.value); // 0

// Write — triggers all subscribers
count.value = 1;

// Untracked read (won't subscribe)
count.peek();
SignatureDescription
State(initial, opts?)Create a signal with initial value
.valueGet/set the current value (tracked read, notifying write)
.peek()Read without subscribing
.binding()Create a Binding wrapper for this signal
.dispose()Clean up all subscribers

Options

OptionTypeDescription
equalsFunctionCustom equality check (default: ===). Return true to skip notification.
nameStringDebug name for the signal

Computed

A read-only derived signal. The function runs lazily and caches its result. Re-evaluates only when a dependency changes.

const count = State(3);
const doubled = Computed(() => count.value * 2);

console.log(doubled.value); // 6
count.value = 5;
console.log(doubled.value); // 10
SignatureDescription
Computed(fn)Create a derived signal from fn
.valueRead the computed value (tracked, lazy evaluation)
.peek()Read cached value without subscribing
.dispose()Stop tracking dependencies

Effect

Runs a side-effect function immediately and re-runs it whenever any tracked dependency changes.

const name = State('Alice');

// Runs immediately, re-runs when name changes
const fx = Effect(() => {
  document.title = `Hello, ${name.value}`;
});

name.value = 'Bob'; // title updates to "Hello, Bob"

// Clean up when done
fx.dispose();
SignatureDescription
Effect(fn)Create a reactive side effect
.dispose()Stop tracking and clean up

Binding

Two-way value bridge with optional read/write transforms. Used to connect controls to state with data transformation.

const celsius = State(0);

// Read transform: C → F, Write transform: F → C
const fahrenheit = Binding(
  celsius,
  c => c * 9/5 + 32,    // read: celsius → fahrenheit
  f => (f - 32) * 5/9   // write: fahrenheit → celsius
);

console.log(fahrenheit.value); // 32
fahrenheit.value = 212;
console.log(celsius.value);    // 100
SignatureDescription
Binding(source, transform?, inverse?)Create a two-way binding
.valueGet/set through the transforms
.get()Alias for reading .value
.set(v)Alias for writing .value

Store

Deep reactive object with path-based access. Granular subscriptions at any nesting level.

const store = Store({
  user: { name: 'Alice', age: 30 },
  items: ['apple', 'banana']
});

// Path-based read (tracked)
store.get('user.name');       // 'Alice'
store.get('items[0]');        // 'apple'

// Path-based write (notifies subscribers of that path)
store.set('user.name', 'Bob');

// Read-modify-write
store.update('user.age', age => age + 1);

// Delete a key
store.delete('user.age');

// Whole-object access
const snapshot = store.value;
store.peek(); // untracked full read
MethodDescription
Store(initial)Create a deep reactive store
.get(path)Tracked read at dot/bracket path
.set(path, value)Update value at path, notify subscribers
.update(path, fn)Read-modify-write: fn(currentValue) → newValue
.delete(path)Remove key and notify
.valueGet/set entire object (tracked)
.peek()Untracked full read

Path Syntax

Paths support dot notation and bracket indexing:

store.get('user.name')        // dot notation
store.get('items[0]')         // array index
store.get('user.items[0].id') // mixed

batch

Batch multiple signal updates together. Subscribers are notified only once after the batch completes.

const a = State(1);
const b = State(2);

// Without batch: Effect fires twice
// With batch: Effect fires once with final values
batch(() => {
  a.value = 10;
  b.value = 20;
});

untrack

Read signal values without creating a dependency. Useful inside Effects when you need a value but don't want to re-run when it changes.

const count = State(0);
const label = State('Count');

Effect(() => {
  // Tracks count, but NOT label
  const l = untrack(() => label.value);
  console.log(`${l}: ${count.value}`);
});

Context

Stack-based dependency injection. Provide a value within a scope, consume it anywhere inside.

const ThemeCtx = Context('light');

// Provide a value within a scope
ThemeCtx.provide('dark', () => {
  const theme = ThemeCtx.use(); // 'dark'
  buildDarkUI(theme);
});

// Outside the scope, returns default
ThemeCtx.use(); // 'light'
MethodDescription
Context(defaultValue)Create a context with default
.provide(value, fn)Push value, execute fn, pop
.use()Read current value (or default)
