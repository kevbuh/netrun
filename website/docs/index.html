Getting Started

AetherUI is a SolidJS-style reactive UI framework for building interfaces with declarative, composable views.

What is AetherUI?

AetherUI provides a signal-based reactivity system and a rich set of UI primitives inspired by SwiftUI. It runs in the browser with no build step — plain JavaScript, no bundler required.

Key ideas:

  • Signals — fine-grained reactivity via State, Computed, and Effect
  • Declarative views — composable view tree with modifier chaining
  • Token system — spacing, colors, and radii resolve to CSS custom properties
  • No virtual DOM — direct DOM updates, minimal overhead

Script Loading Order

AetherUI files must be loaded in order. Each file builds on the previous:

<!-- 1. Reactivity primitives -->
<script src="aether/ui/state.js"></script>

<!-- 2. Base View class + modifiers -->
<script src="aether/ui/view.js"></script>

<!-- 3. Layout & text primitives -->
<script src="aether/ui/primitives.js"></script>

<!-- 4. Interactive controls -->
<script src="aether/ui/controls.js"></script>

<!-- 5. Reactive containers -->
<script src="aether/ui/containers.js"></script>

<!-- 6. Overlays (Sheet, Alert, Popover, Menu, Toast) -->
<script src="aether/ui/overlay.js"></script>

<!-- 7. Component API -->
<script src="aether/ui/component.js"></script>

<!-- 8. Main assembly (mount, append, serialize, globals) -->
<script src="aether/ui/aether-ui.js"></script>

<!-- 9. VirtualList (optional) -->
<script src="aether/ui/virtual.js"></script>

After loading aether-ui.js, all primitives are available on window (e.g. VStack, Button, State).

Hello World

const name = State('World');

const view = VStack(
  Text('Hello, AetherUI!').font('title1'),
  HStack(
    Text('Name:'),
    TextField(name, 'Enter name')
  ).spacing(2),
  Text(() => `Hello, ${name.value}!`).bold()
).spacing(4).padding(6);

AetherUI.mount(view, document.getElementById('app'));

Type in the text field and the greeting updates instantly — no re-renders, no diffing.

Reactivity at a Glance

// Signals hold reactive values
const count = State(0);

// Computed values derive from signals
const doubled = Computed(() => count.value * 2);

// Effects run side effects when dependencies change
Effect(() => {
  console.log('Count is:', count.value);
});

// Update the signal — computed + effect re-run automatically
count.value = 5;  // logs "Count is: 5", doubled.value === 10

Modifier Chaining

Every view supports fluent modifier chaining. Modifiers resolve semantic tokens to CSS custom properties:

Text('Styled text')
  .font('headline')
  .foreground('secondary')
  .padding(4)           // → var(--nr-space-4) = 16px
  .background('surface') // → var(--nr-bg-surface)
  .cornerRadius('md')   // → var(--nr-radius-md) = 8px
  .shadow('card')
  .onTap(() => alert('Tapped!'))
