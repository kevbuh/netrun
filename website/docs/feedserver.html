Feed Server

A standalone Go microservice serving ranked feed data on port 8400, with its own SQLite database and auto-refresh.

Overview

The feed server is a separate Go process at feedserver/. It is independent of the main Electron app — has its own SQLite database (feedserver/feedserver.db) and runs as an HTTP server. The frontend connects to it directly via http://localhost:8400.

Not routed through IPC. Unlike the rest of the backend, the feed server is accessed directly via HTTP from the renderer, not through the Electron IPC bridge.

Startup Sequence

1. store.New(cfg.DBPath)         → open SQLite DB
2. Seed sources from SeedCatalog → initial source list
3. fetch.NewFetcher(db, catalog) → create fetcher
4. goroutine: fetcher.RefreshAll()       → immediate fetch
5. goroutine: ticker every 10 minutes    → periodic refresh
6. api.NewServer(db, fetcher, catalog)   → HTTP server
7. http.ListenAndServe(:8400)            → start serving

Configuration

SettingDefaultOverride
Port8400--port flag or FEEDSERVER_PORT env var

REST API

Timeline & Content

MethodPathDescription
GET/api/timelineRanked feed items. Query: sort, category, search, limit (default 100), offset
POST/api/refreshTrigger immediate refresh. Returns { fetched: n }
GET/api/rss-proxyProxy RSS fetch via ?url=... (15s timeout)

Sources

MethodPathDescription
GET/api/sourcesList all feed sources
POST/api/sourcesAdd a new source (triggers background refresh)
PUT/api/sources/{key}/toggleEnable/disable a source
POST/api/sources/syncBulk-sync source enabled/disabled preferences

User Actions

MethodPathDescription
POST/api/readMark item as read. Body: { link }
POST/api/saveSave/bookmark an item
DELETE/api/saveUnsave an item
POST/api/hideHide an item from the feed
POST/api/rateRate an item. Body: { link, rating }
GET/api/savedList saved items

State & Config

MethodPathDescription
GET/api/stateUser state (enabled sources, preferences)
PUT/api/rank-paramsUpdate ranking algorithm parameters

Internal Packages

PackagePurpose
internal/apiHTTP handlers and routing
internal/fetchRSS/feed fetching and parsing
internal/modelData types: Source, RankParams, etc.
internal/rankFeed ranking algorithm
internal/storeSQLite persistence layer

CORS

The withCORS middleware wraps the HTTP mux, allowing all origins with GET, POST, PUT, DELETE, OPTIONS methods. This permits the Electron renderer (loaded from localhost:8000) to call the feed server at localhost:8400.

Build & Run

# Build Go binary → dist/feedserver
npm run feedserver:build

# Build + start both feedserver and Electron
npm run feedserver:start

# Development mode (go run, no build)
npm run feedserver:start-dev

Frontend Integration

// src/js/feed.js connects directly to the feed server
const response = await fetch('http://localhost:8400/api/timeline?limit=50');
const items = await response.json();

// Trigger refresh
await fetch('http://localhost:8400/api/refresh', { method: 'POST' });

// Save an item
await fetch('http://localhost:8400/api/save', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ link: item.link }),
});
