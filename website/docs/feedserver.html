Feed Engine

An integrated TypeScript feed system built into the core backend, using the main better-sqlite3 database and communicating via IPC.

Overview

The feed engine is fully integrated into the Electron core backend at src/core/. Unlike the previous standalone Go microservice, it shares the main better-sqlite3 database and is accessed by the frontend through Electron IPC — the same channel used by every other subsystem.

No separate process. The feed engine runs in-process with the Electron main process. There is no HTTP server, no separate port, and no CORS configuration needed.

Startup Sequence

1. registerFeedsIPC()              → register all db:feed-* handlers
2. feedStore.listSources()         → load sources from DB
3. fetcher.setSources(sources)     → configure fetcher
4. fetcher.startBackgroundRefresh()
   ├── immediate refreshAll()      → fetch stale sources
   └── setInterval(10 min)         → periodic refresh

Fetcher

The fetcher (src/core/feeds/fetcher.ts) handles data acquisition from multiple source types with a concurrency-limited semaphore (max 10 parallel fetches).

Source Types

SpecialHandlerDescription
arxivfetchArxivItems()arXiv CS RSS feed, extracts paper IDs
hnfetchHNItems()Hacker News best stories API (top 30)
polymarketfetchPolymarketItems()Polymarket breaking/biggest-movers via SSR data
(default)fetchRSSItems()Generic RSS/Atom feed parsing

Staleness

Before fetching, the fetcher checks each source's last fetch timestamp. Sources are only re-fetched if older than 10 minutes (600 seconds). This prevents redundant network requests when data is still fresh.

Ranking Algorithm

The ranking engine (src/core/feeds/rank.ts) supports three sort modes:

SortBehavior
foryou (default)Personalized scoring based on interest profile, source affinity, recency, and exploration bonus. Results are interleaved across categories.
latestChronological by publication date (newest first)
citationsBy citation count or HN score (highest first)

For-You Scoring

The for-you algorithm combines several signals:

  • Content score — matches title tokens and categories against an interest profile built from read, saved, and rated items
  • Source affinity — engagement rate per source (reads, saves, ratings vs. total items)
  • Recency — bonus for items published within the last 20 hours, scaled by weightRecency
  • Exploration — bonus for items from low-affinity sources to promote diversity

Rank Parameters

Users can tune the ranking weights via db:feed-rank-params:

ParameterDefaultPurpose
weightBase0.7Base multiplier for content score
weightAffinity0.3Source affinity contribution
weightRecency1.0Recency bonus scale
weightExploration0.10Exploration bonus for underread sources
maxPerCategoryRun3Max consecutive items from one category before interleaving

Frontend Integration

// src/js/feed.js communicates via IPC
const { items, total } = await electronAPI.invoke('db:feed-timeline', {
  sort: 'foryou', limit: 50
});

// Trigger refresh
const { fetched } = await electronAPI.invoke('db:feed-refresh');

// Save an item
await electronAPI.invoke('db:feed-save', item.link);

// Mark as read
await electronAPI.invoke('db:feed-mark-read', item.link);

// Update rank parameters
await electronAPI.invoke('db:feed-rank-params', {
  weightBase: 0.7, weightAffinity: 0.3,
  weightRecency: 1.0, weightExploration: 0.10,
  maxPerCategoryRun: 3,
});
