Components

Reusable functional components with automatic disposal scoping. Define once, instantiate anywhere.

Component

Register a named component. The render function receives props and children, returns a View. Any State, Computed, or Effect created inside the render function is automatically disposed when the component's view is disposed.

const Counter = Component('Counter', (props, children) => {
  const count = State(props.initial || 0);

  return VStack(
    Text(() => `Count: ${count.value}`).font('title2'),
    HStack(
      Button('-').onTap(() => count.value--),
      Button('+').onTap(() => count.value++)
    ).spacing(2),
    ...children
  ).spacing(3).padding(4);
});

Usage

// With props
const counter = Counter({ initial: 10 });

// With children
const counter = Counter({ initial: 0 },
  Text('Extra content below the buttons')
);

// No props, just children
const counter = Counter(
  Text('Child view')
);

How Props Work

The first argument is treated as props if it's a plain object (not a View). Otherwise, all arguments are treated as children.

// props = { initial: 5 }, children = [Text('Hi')]
Counter({ initial: 5 }, Text('Hi'))

// props = {}, children = [Text('Hi')]
Counter(Text('Hi'))

Disposal Scoping

Signals and effects created inside a component's render function are tracked and automatically disposed when the component's root view is disposed. This prevents memory leaks.

const Timer = Component('Timer', () => {
  const elapsed = State(0);

  // This effect is scoped to the component.
  // It will be cleaned up when the Timer view is disposed.
  const interval = setInterval(() => elapsed.value++, 1000);

  const view = Text(() => `${elapsed.value}s`).font('title1');

  view.onDisappear(() => clearInterval(interval));
  return view;
});

defineComponent

Alias for Component (backward compatibility).

const MyWidget = defineComponent('MyWidget', (props) => {
  return Text(props.label);
});

Registry

Components are registered globally by name. You can look them up or list all registered components.

// Look up a registered component
const CounterFactory = getComponent('Counter');
if (CounterFactory) {
  const c = CounterFactory({ initial: 42 });
}

// List all registered component names
const names = listComponents();
// ['Counter', 'Timer', 'MyWidget']
FunctionDescription
Component(name, renderFn)Register and return a component factory
defineComponent(name, fn)Alias for Component
getComponent(name)Look up a registered component by name (or null)
listComponents()Return array of all registered component names

Worked Example

A reusable card component with header, body, and optional footer:

const Card = Component('Card', (props, children) => {
  return VStack(
    // Header
    HStack(
      Text(props.title).font('headline'),
      Spacer(),
      props.action
        ? Button(props.action.label).ghost().small()
            .onTap(props.action.handler)
        : null
    ),

    // Body
    ...children

  ).spacing(3)
   .padding(4)
   .background('raised')
   .cornerRadius('lg')
   .shadow('card');
});

// Usage
Card({ title: 'Recent Activity', action: { label: 'View All', handler: viewAll } },
  ForEach(activities, a => a.id, a =>
    HStack(Text(a.label), Spacer(), Text(a.time).foreground('tertiary'))
  ).spacing(2)
)
Tip Components set a data-component attribute on the root element, which is useful for debugging and styling.
