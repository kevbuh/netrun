Containers

Reactive containers for conditional rendering, dynamic lists, and structural grouping.

ForEach

Reactive list renderer. When the items signal changes, views are efficiently reconciled using an optional key function.

const items = State(['Apple', 'Banana', 'Cherry']);

// Simple (no key — full rebuild on change)
ForEach(items, item => Text(item))

// With key function (efficient reconciliation)
ForEach(items, item => item, item =>
  HStack(
    Text(item),
    Spacer(),
    Button('Remove').ghost().small()
  )
).spacing(2)
SignatureDescription
ForEach(items, renderFn)No key — rebuilds all on change
ForEach(items, keyFn, renderFn)Keyed — only adds/removes changed items
ModifierDescription
.spacing(s)Gap between items (space token)
.itemTransition(config)Animate item enter/exit. Preset: 'fade', 'slide', 'scale' or custom config.

Item Transitions

// Named preset
ForEach(items, i => i.id, renderItem)
  .itemTransition('fade')

// Custom transition
ForEach(items, i => i.id, renderItem)
  .itemTransition({
    in: { opacity: [0, 1], y: [-20, 0], duration: 200 },
    out: { opacity: [1, 0], duration: 150 }
  })

Available presets: 'fade' (opacity), 'slide' (opacity + translateY), 'scale' (opacity + scale). Each preset automatically staggers items on insert.

List

Styled wrapper around ForEach with dividers and inset support.

List(items, item => item.id, item =>
  HStack(
    Text(item.name),
    Spacer(),
    Text(item.date).foreground('tertiary').font('footnote')
  ).padding(3)
).dividers().inset()
ModifierDescription
.dividers()Show dividers between items
.inset()Add padding around the list

Group

Logical grouping with display: contents — groups children without adding a DOM wrapper that affects layout.

VStack(
  Group(
    Text('A'),
    Text('B')
  ),
  Text('C')
).spacing(2) // gap applies uniformly to A, B, C

Section

Header + content grouping.

Section('Account Settings',
  FormField('Name', TextField(name)),
  FormField('Email', TextField(email))
).spacing(3)

// Custom header view
Section(
  Text('Advanced').font('headline').foreground('secondary'),
  Toggle(debug, 'Debug mode')
)
ModifierDescription
.spacing(s)Gap in the content area

Show

Conditional rendering. Shows or hides content based on a signal. Content is created/destroyed, not just hidden.

const isLoggedIn = State(false);

Show(isLoggedIn,
  () => Text('Welcome back!'),
  () => Button('Log in').primary()
)

// With transition
Show(isVisible,
  () => DetailPanel()
).transition('fade')

// Lazy rendering (deferred via requestIdleCallback)
Show(isReady,
  () => HeavyComponent()
).lazy()
ModifierDescription
.transition(config)Animate show/hide. Preset: 'fade', 'slide', 'scale'
.lazy()Defer rendering via requestIdleCallback

Switch

Multi-branch conditional rendering. Like a switch statement for views.

const page = State('home');

Switch(page, {
  home:     () => HomePage(),
  settings: () => SettingsPage(),
  profile:  () => ProfilePage(),
  default:  () => Text('Not found')
}).transition('fade').lazy()
ModifierDescription
.transition(config)Animate branch transitions
.lazy()Defer rendering

EmptyState

Centered placeholder for empty content areas.

EmptyState({
  icon: 'inbox',
  title: 'No messages',
  message: 'Your inbox is empty.',
  action: {
    label: 'Compose',
    handler: () => openCompose()
  }
})
OptionDescription
iconIcon name string
titleTitle text
messageDescription text
action{ label, handler } — optional action button
