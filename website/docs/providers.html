LLM Providers

Two LLM providers — Ollama (local) and OpenRouter (cloud) — managed by a central registry with automatic default selection.

Provider Registry

The ProviderRegistry singleton manages provider registration and selection. Located at src/core/providers/registry.ts.

MethodDescription
register(provider)Register a provider (first registered = default)
get(name)Get a provider by name
getDefault()Get the current default provider
setDefault(name)Set active provider (throws if not registered)
names()List all registered provider names

Built-in Providers

Ollama (Local)

Local LLM inference via the Ollama API. Registered first, making it the default provider. Connects to http://localhost:11434.

  • No API key required
  • Models run locally on the user's machine
  • Supports both chat() and chatStream()

OpenRouter (Cloud)

Cloud LLM access via the OpenRouter API. Provides access to many models (GPT-4, Claude, Gemini, etc.).

  • Requires an API key (stored encrypted via safeStorage)
  • API key stored in both Electron's encrypted storage and the settings DB
  • Supports both chat() and chatStream()

Registration at Startup

Providers are registered during initCore():

// src/core/init.ts
providerRegistry.register(ollamaProvider);    // first = default
providerRegistry.register(openrouterProvider);

// Restore saved preference from settings DB
const savedProvider = settings.get('aiProvider');
if (savedProvider) providerRegistry.setDefault(savedProvider);

// Restore API key (migration path)
const savedKey = settings.get('openrouterApiKey');
if (savedKey) openrouterProvider.setApiKey(savedKey);

How Agents Use Providers

The agent runtime gets the default provider and calls its LLM methods:

// src/core/agents/runtime.ts
const provider = providerRegistry.getDefault();

// Non-streaming call (tool call loop)
const response = await provider.chat(messages, tools, {
  toolChoice: 'auto',
});

// Streaming call (final response)
for await (const chunk of provider.chatStream(messages)) {
  yield { type: 'token', content: chunk.content };
}

Frontend IPC

The frontend manages providers through electronAPI:

// List available providers
const providers = await electronAPI.providerList();
// → ['ollama', 'openrouter']

// List models for a provider
const models = await electronAPI.providerModels('ollama');
// → ['llama3.1', 'codellama', ...]

// Switch the active provider
await electronAPI.providerSetDefault('openrouter');

// Store an API key (encrypted)
await electronAPI.providerSetApiKey('openrouter', 'sk-or-...');

// Get stored key
const key = await electronAPI.providerGetApiKey('openrouter');

Provider Interface

Each provider implements these methods:

MethodDescription
chat(messages, tools?, options?)Non-streaming LLM call, returns full response
chatStream(messages, options?)Streaming LLM call, returns async iterator of chunks
listModels()List available models
setApiKey(key)Set the API key (cloud providers only)
