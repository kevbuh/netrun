Database

better-sqlite3 with WAL mode, a prepared statement cache, and IPC-routed query handlers.

Connection

The database connection is a singleton in src/core/db/connection.ts. It opens on first access and applies pragmas:

// Database location
const dataDir = process.env.ARXIV_DATA_DIR
  ?? app?.getPath('userData')
  ?? process.cwd();
const dbPath = path.join(dataDir, 'netrun.db');

// Pragmas applied on open
db.pragma('journal_mode = WAL');
db.pragma('busy_timeout = 5000');
db.pragma('foreign_keys = ON');
ExportDescription
getDb()Get the database connection (lazy init)
prepare(sql)Get a cached prepared statement
closeDb()Close connection and clear cache (called on app quit)

Statement Cache

The prepare(sql) function caches prepared statements by SQL string, avoiding re-preparation overhead:

// First call: prepares and caches
const stmt = prepare('SELECT * FROM users WHERE google_id = ?');
// Subsequent calls: returns cached statement
const stmt2 = prepare('SELECT * FROM users WHERE google_id = ?');
// stmt === stmt2

Query Handler Pattern

IPC handlers are registered in src/core/ipc/db-queries.ts. Each handler maps a db:* channel to a query function:

// src/core/ipc/db-queries.ts
import { ipcMain } from 'electron';
import * as userQueries from '../db/queries/users';

export function registerDbQueriesIPC() {
  ipcMain.handle('db:user-get', (_event, googleId) => {
    return userQueries.getUser(googleId);
  });

  ipcMain.handle('db:user-upsert', (_event, data) => {
    return userQueries.upsertUser(data);
  });

  // ... 60+ more handlers
}

// The dbQuery shortcut prefixes with 'db:'
const user = await electronAPI.dbQuery('user-get', googleId);
const events = await electronAPI.dbQuery('calendar-list', googleId);
await electronAPI.dbQuery('highlight-save', { url, page_num, rects_json, color });

Migrations

Schema migrations are inline ALTER TABLE statements wrapped in try/catch for idempotency. New columns are added without breaking existing databases:

// Idempotent migration pattern
try {
  db.exec('ALTER TABLE context_meta ADD COLUMN file_type TEXT DEFAULT "topic"');
} catch (e) {
  // Column already exists — ignore
}
